## 一个反直觉的发现

我的 S6F 策略用了一条简单条件:

```sql
WHERE (today_close / 60_days_ago_close - 1) * 100 < -15  -- 近 60 日跌 >15%
```

这条件看起来无害 —— 跌超 15% 就算"深跌",可以作为"低位反转"候选。

直到我用前复权价 (close_qfq) 重新跑了一遍回测。

## 发现 14 只"假深跌股"

**2026-05-07 对比**:

| 口径 | 触发数 |
|---|---|
| close (不复权) | 901 只 |
| **close_qfq (前复权)** | 887 只 |
| 只 raw 触发 (**假深跌**) | **14 只** |
| 只 qfq 触发 (raw 漏掉) | 0 只 |

这 14 只股在 close 看起来跌了 15-35%,前复权后实际只跌 0-12%。**它们不是跌,是除权了**。

## 极端案例 · 华海诚科

688535 华海诚科 · close 跌 21%, qfq 涨 17%, **差了 38 个百分点**。

怎么回事?2025 年 **10 送 15 股高送转** + 分红 1 元 / 股。除权后价格跳水,看起来"跌了 40%",但前复权价跟除权日的前一天对齐,**还原了真实走势**。

**珠城科技 / 美能能源** 类似,都是近 60 日内有过高送转 / 大额分红。

## 为什么 S6F 没受影响

**好消息**: 我的 S6F 策略除了 "近 60 日跌 >15%",还叠加了:

1. **PE < 20** — 很多除权假跌股 PE 不够低
2. **市值 >= 80 亿** — 上面 14 只中有一半 < 60 亿被剔
3. **ROE >= 8 且向上** — 高送转股未必财务指标向上
4. **非黑名单行业** — 燃气 2 只直接被剔

**坏消息**: 如果你的策略只用 "深跌" 做单条件筛选,**14 只假深跌就是你的候选池的 1-2%**。

## 更狠的坑 · 后复权的反向问题

前复权解决历史数据,后复权反过来:
- **后复权**: 把历史价格调低,让今日价格 = 真实除权后价
- 用于**当日实盘价格对齐**,比如你要算"相对 N 日前的涨幅百分比"

**多数 A 股量化项目用前复权**,因为对策略友好。

## 实操建议

### 1. 拉数据就用 akshare `adjust='qfq'`

```python
# 错的
df = ak.stock_zh_a_hist(symbol='601336', period='daily')

# 对的
df = ak.stock_zh_a_hist(symbol='601336', period='daily', adjust='qfq')
```

### 2. 已经入库的数据, 加列不删库

我的做法是 `daily` 表加一个 `close_qfq` 列,原 close 保留。这样:
- 新策略用 close_qfq
- 老策略不动
- 能同时对比两种口径的回测结果

回填脚本 5500 股 × 3 年本地跑了 45 分钟。核心是循环 code 调 akshare 的 qfq 接口拉历史,拉完写 `close_qfq` 列回去,失败代码走 retry 队列。

### 3. 定期增量更新

除权通常发生在每年 **5-7 月**(半年报前)和 **12-1 月**(全年报前)。

我的 `run_daily.bat` 每晚增量拉最近的 close_qfq,所以当日除权的股第二天自动修正。

## 结论 · 一条简单的调整,可能影响你 1-2% 的候选池

不复权和前复权在 95% 的股票上差不多,但那 5% 系统性偏差,在"深跌 >15%"这种条件下 **放大 10 倍**。

**如果你的策略用了"近 N 日涨跌幅"作为筛选**,请考虑:

1. 现在就换 close_qfq
2. 或者在原有筛选后加一层"过去 60 日无除权除息"

## 附录 · 快速自检

在你的 sqlite 里跑一下:

```sql
-- 近 60 日 close_qfq / close 偏差 > 5% 的股
SELECT code, COUNT(*) as n
FROM daily
WHERE date >= date('now', '-60 days')
  AND close > 0 AND close_qfq > 0
  AND ABS((close_qfq - close) / close * 100) > 5
GROUP BY code
ORDER BY n DESC;
```

如果某代码 n > 30, 基本就是近期除权过的,你的历史数据里"深跌"筛选对它就是假信号。
