## 一句话总论

注册海外服务(Oracle Cloud / Porkbun / Cloudflare 某些业务等)之前,**第一步不是填表,是测你的公网出口 IP 走的是什么段**。

选错出口 → 扣款过了但 tenancy 静默失败 / Veriff 拒签中国身份证 / 账号莫名被 hold 待审核。

## 我踩的一次坑

连续几晚在公司笔记本上尝试注册 Oracle Cloud 免费层。

第一晚:账号创建,信用卡预授权 $1 过了,结果 `tenancy` 静默失败,后台连登不进。
第二晚:换卡重试,预授权又扣一次 $1,tenancy 仍然失败。
第三晚(家里):用同一邮箱 + 同一信用卡,秒开,正常。

**唯一变量**:出口 IP。

## 根因:办公网络出口往往是企业段

很多互联网公司的办公网会把员工流量聚合到**企业级 VPN 出口**,通常是 HK/美国/新加坡的机房 IP,对应某个具体的 **ASN**(AS 号)+ 企业名。

举例:

- 地理上看像是 HK(country=HK, city=HK)
- 实际 ASN 属于"某公司 H.K. Limited"这种企业段
- **中国身份证 + 香港企业 IP** 对海外风控来说 = 身份与位置不一致

即使你把 Cloudflare WARP 之类的 VPN 关了,**底层办公网出口本身**就是企业段。

这个"脏"不是违法意义上的脏 — 是海外服务的 KYC/反欺诈系统把企业段当**代理/跳板**处理,默认风险高。

## 诊断命令:多源交叉

**不要只用一个源**,企业内网可能劫持流行的 IP 查询域名:

```bash
curl -s https://ipinfo.io/json        # 可能被内网 DNS 劫持
curl -s https://ifconfig.me           # 同上
curl -s https://icanhazip.com         # 通常真实
curl -s https://myip.ipip.net         # 国内源,真实
```

为什么要多源:我之前发现有个公司的 VPN 客户端**选择性劫持** ipinfo.io(把"异地检测类"域名标记为走审计通道),但 icanhazip.com 这类不太出名的走的是家宽直连。结果 ipinfo.io 报 HK 段,icanhazip.com 报真实家宽 IP。如果只看一个,诊断会错。

**关键字段**:不要只看 `country`,要看 `org`(ASN + 公司名)。

## 典型 ORG 字段识别

| org 字段示例 | 性质 | 海外服务接受度 |
|---|---|---|
| `AS4134 Chinanet-Backbone` | ✅ 中国电信家宽 | 高 |
| `AS4837 CHINA UNICOM` | ✅ 联通家宽 | 高 |
| `AS56040 China Mobile` | ✅ 移动 | 高 |
| `AS63855 ... H.K. Limited` | 🔴 企业 VPN 段 | **极低** |
| `AS13335 Cloudflare` | 🔴 WARP/CDN | 低 |
| `AS14061 DigitalOcean` | 🔴 机房 | 极低 |
| `AS16509 Amazon.com` | 🔴 AWS 机房 | 极低 |

**规则**:只要 ORG 带 `Limited` / `Cloud` / `Hosting` / CDN 厂商名,大概率都会被海外 KYC 扣分。

## 换出口的排序(干净到脏)

| 出口 | 干净度 | 成本 | 备注 |
|---|---|---|---|
| 🥇 手机 4G 热点 | ⭐⭐⭐⭐⭐ | 流量钱 | 关 WiFi → 开个人热点 → 电脑连;中国三大运营商手机段接受度最高 |
| 🥈 家宽直连 | ⭐⭐⭐⭐ | 0 | 关所有 VPN,让机器走家里 ISP |
| 🥉 咖啡厅 WiFi | ⭐⭐⭐ | 一杯咖啡 | 通常是企业宽带,比公司 VPN 干净 |
| ❌ 公司办公网 | ⭐ | 0 | 大厂基本都走企业段,一注册就扣分 |
| ❌ Cloudflare WARP | ⭐ | 0 | CF IP 段,海外服务直接拒 |
| ❌ 商业 VPN | ⭐ | $ | 机房 IP,比办公网还差 |

**实测最稳**:回家开手机热点,短短几分钟连上电脑完成注册,关掉热点。这是成本最低的"一次性干净出口"方案。

## 各家海外服务的敏感度

根据我过去几个月的实测,主流服务的 KYC 严格度大致是:

| 服务 | KYC 工具 | HK 企业段触发 | 中国身份证友好 |
|---|---|---|---|
| Oracle Cloud Free | (无 KYC,走风控) | 🔴 几乎必挂 | 不相关 |
| Porkbun | Veriff | 🔴 常触 | 较差 |
| Namecheap | Jumio/Sumsub | 🟡 有时触 | 中等 |
| Spaceship | - | 🟡 未知 | 较好 |
| Cloudflare | - | 🟢 宽松 | 好 |

**观察**:越是小众/反欺诈严格的服务(Oracle、Porkbun),IP 干净度越重要;Cloudflare 这种面向全球开发者的服务相对宽松。

## 一些看似反直觉但其实有道理的事

**问**:为什么公司都是大公司,出口 IP 反而比家宽脏?

**答**:海外风控系统认的是"这个 IP 段通常用来干什么"。大公司办公网出口 IP 历史上会有:

- 不少员工用公司 VPN 访问海外服务,已被各家标记
- 企业流量聚合,异常模式多
- VPN 类厂商有时会租用这些段做代理

所以风控模型里,企业办公网出口 = "代理+跳板"模式。反倒是你家里的家宽 ISP 段干净,因为它不是代理出口。

## 一键检查模板

注册任何海外服务前,跑这三条命令,10 秒搞定:

```bash
# 1. 查真实出口
curl -s https://icanhazip.com

# 2. 查 ORG(判断是否企业段)
curl -s https://ipinfo.io/$(curl -s https://icanhazip.com)/json | grep -E '"(org|country|city)"'

# 3. 如果 org 里有 Limited/Cloud/Hosting/企业名,换手机热点再注册
```

## 一句话收尾

如果你是国内开发者要注册海外服务,**办公网络从来不是合适的场所**。默认回家或开手机热点,10 秒诊断一下 IP ORG,5 分钟完成注册。不要省那 5 分钟,事后修复 tenancy 卡死 / 账号 hold 的成本是几十倍。
