## 为什么要写这个

起名看似感性,其实是个结构化决策问题。我最近给自己的工作室 (Laither) 和 3 个产品线起名,发现用同一套流程跑下来又快又不后悔。

这篇把这套流程沉淀成 5 步方法论,适用于:

- 工作室 / 个体户 / 公司主品牌
- App / SaaS / 子产品
- 既有产品的 rebranding

**不适用**:域名抢注式起名 (3 字母短域名那种)、个人 KOL 品牌 (那类逻辑不同)。

---

## Step 1 · 先定调性,再谈名字

起名失败最常见的原因是一上来就开始堆候选词。你会很快陷入"每个都有点意思又都不完美"的疲劳区。

**应该先让自己回答 4 个正交问题**:

1. **调性** (名字外观语感) · 多选
   - 技术极客感 / 中文意境 / 中英双轨 / 抽象符号
2. **身份感** (品牌气质) · 多选
   - 独立工匠 / 数据理性 / 生活秩序 / 智能未来
3. **范畴内核** (所有产品的共同母题) · 单选
   - 数据分析 / 工具效率 / 复盘内视 / 工匠积累
4. **个人印记** · 单选
   - 不带 / 带姓氏首字母 / 全真名

四个问题答完,你会发现自己想要的名字长什么样已经具象了一半。

## Step 2 · 看似矛盾的多维度是机会,不是问题

我当时选了"中文意境 + 中英双轨 + 抽象符号"三种调性,四种身份感全要。发散对吗?

不是。这不是并列选项,而是**品牌的三个层次**:

- **调性** = 怎么"看起来 / 读起来"
- **身份感** = 怎么"感觉像谁"
- **范畴** = "所有产品在讲同一个故事"

好名字能同时打三层。举几个大家熟悉的:

- **Notion** → 抽象符号 + 智能工匠 + 思考知识
- **知乎** → 中文意境 + 智能独立 + 求知
- **豆瓣** → 中英双轨 + 独立生活 + 文艺生活

破解"发散"的方法是**用一个核心意象投射到多层**。我最后选的 Laither 的核心意象是"分层积累",这个意象同时打了抽象符号 (Layer 词根)、独立工匠气质、工匠积累内核。

## Step 3 · 从内核出发造词,不从词库选词

**不要**翻开 10000 个英文单词表挑"好听的"。那会越挑越累,而且你挑到的每一个词都已经被别人想过无数次了。

**应该**确定内核,用 2-3 个核心动词/词根融合造新词。

### 我当时的造词路径

内核 = **复利 / 积累 / 层叠 / 沉淀**

核心动词:
- Gather (收拢) → Gither (去 a 变硬)
- Layer (分层) → Layar / Lither
- Stack (堆) → Stacc / Stackly
- Fold (叠) → Foldry / Cofold
- Accrue (复利) → Accrur

融合造词:
- Layer + Gather → **Laither** ← 最后选这个
- Layer + Gither → Lither (.com 15 年前被投资者占)
- Gather + Lay → Gathlay / Githlay
- Layer + Pile → Laypile

### 造词经验规则

| 规则 | 为什么 |
|---|---|
| 5-8 字母最佳 | 短于 5 字母 .com 基本都被占;9+ 字母难记 |
| 避免两个爆破音相连 | `Gith-lay` 读起来别扭,`Lai-ther` 顺 |
| 避免三连元音/辅音 | 外国人读不出来 |
| 词尾 -er / -or / -ly / -ry 常加分 | 行为化/地点化,抽象品牌友好 |
| 能配中文 2 字 | Laither 配"叠时"/"拾微",有中英双轨落地空间 |
| `.com` 空闲是刚性前提 | 连不上 `.com` 就放弃,别谈 `.io` / `.app` 替代 |

## Step 4 · 域名 / 搜索 / 商标三查一

**优先级**:域名 > 搜索占用 > 商标

### 4.1 域名查 (免费 · 快)

用 RDAP 官方查询,不要用 `whois.com` 之类的站 (被 Cloudflare 卡验证码):

```bash
curl -s "https://rdap.verisign.com/com/v1/domain/LAITHER.COM"
# 404 = 可注册 ✅
# 200 返回注册信息 = 已占 ❌
```

批量查法:准备 5-10 个候选,并行跑 RDAP 过滤:

```bash
for n in laither accrur githlay laypile foldry; do
  code=$(curl -s -o /dev/null -w "%{http_code}" "https://rdap.verisign.com/com/v1/domain/${n^^}.COM")
  echo "$n.com : $code"
done
```

`.io` / `.app` 查询换成 `rdap.nic.io` / `rdap.nic.app`。

### 4.2 搜索占用查

用 `cn.bing.com` 或 `duckduckgo.com`,避免 `google.com` 在国内的重定向折腾。

查 `"XXXX"` 带引号强制精确匹配。**关键看前 20 个结果有没有**:

- 知名公司 / 热门产品撞名
- 俚语负面含义 (Urban Dictionary 查一下)
- 大品牌的子产品线用同名

### 4.3 商标查 (可延后,但不能跳过)

- **中国**:商标局 `sbj.cnipa.gov.cn` 查第 9 类 (软件) + 第 42 类 (SaaS 服务)
- **国际**:WIPO `branddb.wipo.int` 或 USPTO TESS

Phase 0 起步阶段不一定立刻注册商标 (¥500+),**但必须确认没有在先商标**。未来商业化跑到稳定营收再正式注册。

## Step 5 · 4 维打分拍板,别凭感觉

剩下 2-3 个候选,用量化打分拍板。

| 维度 | 权重 | 含义 |
|---|---|---|
| 好记好念 | 25% | 用户 5 秒内能拼出来吗?陌生人听一次能复述吗? |
| 多品类装载力 | 25% | 能装下现有+未来 6-12 个月想做的产品吗? |
| 品牌独特性 | 20% | 搜索结果空白度?撞知名品牌的风险? |
| 可商业化 | 15% | 未来转公司/海外/融资时好不好拓展? |
| 中文可搭 | 15% | 能配 2 字中文品牌名吗?不是生硬音译? |

每项 1-10 分打,加权求和。我最后的分数:Laither 9.2 / Accrur 7.55 / Githlay 7.35。**分差超过 1.5 就别犹豫**。

## Step 6 · 拍板后立刻落地 5 件事

选完名字 24 小时内必须做完:

1. **注册 `.com`** — 域名投资 AI 会 7×24 扫 RDAP,犹豫一天真的可能被抢。支付宝能付的 registrar 就 Porkbun / Spaceship
2. **备选域名收藏** — `.io` / `.app` 同名加进 Watchlist,不着急注但盯着
3. **Bundle ID 占坑** — iOS / Android 未来注册 Developer 时用 `com.XXX.*` 前缀先占好
4. **统一文档** — 所有现有笔记里"待定"的占位符全部批量替换
5. **子域规划** — `stock.XXX.com` / `vault.XXX.com` / `blog.XXX.com` 提前想好,免得未来爬来爬去改 DNS

## 5 个反模式 (都踩过一次就长记性)

### ❌ 一上来问"你想叫什么"

对方会答"我也不知道,你推荐"。没定调性前甩 10 个词只会越挑越糊涂。

### ❌ 太早给候选词

"我觉得 Laither 不错你觉得呢?" — 这句话一出,对方的独立判断就结束了。**先维度,再候选,再打分**,顺序不能乱。

### ❌ 迷信"绝对独特"

Githlay 在独特性维度 10/10,但好记好念只有 5/10。**独特性不是越高越好,是"够用的独特"**。

### ❌ 用真名 / 姓氏首字母

带个人印记的品牌未来的问题:

- 转公司要改名 (重建品牌资产)
- 招人后团队不会认同
- 融资时 VC 会劝改名

**品牌资产要能转让**,从第一天就规划好。

### ❌ 只查 .com 不查搜索和商标

有人抢了 "Gemini" 做 APP,Google 起 Gemini AI 直接碾压 — 小品牌撞大公司商标就是白送。

## 时间预算

从零到落地一个好品牌名,在决策框架清晰的前提下大概:

- Step 1-2 定调性 + 挖内核 · 30 min
- Step 3 造词 · 30-60 min
- Step 4 查占用 (5 候选并行) · 10 min
- Step 5 打分拍板 · 10 min
- Step 6 落地 · 30 min

**全流程 2-3 小时**,分散到半天节奏最舒服。

## 后话

这套方法论最大的价值不是"保证起到好名字",而是**保证 3 个月后不后悔**。凭感觉选的名字上线一个月就觉得"好像有点怪",6 个月要换,每次换都丢一截品牌资产。

过一遍 5 步之后,即使最后选出来的名字不是"惊艳",也会是"经得起推敲 + 未来能扩"的稳健解。稳健 > 惊艳,这是我起过几次名字后最真实的体会。
